# Amio Chat: Android Implementation & Credentials Guide

This document outlines the API credentials you need to gather to build the new **Android version** of Amio Chat (replacing the web app version), along with the recommended native Android app project structure.

---

## 1. Credentials Checklist (What to Gather)

To support Google Accounts (OAuth 2.0 Identity), Google Chat API (Gmail), and Firebase Database sync within a native Android app, you will need the following credentials:

### A. Firebase configuration file for Android:
* **`google-services.json`**: This is a configuration file that Android applications use to connect to Firebase services.
  * **How to get it:**
    1. Go to the [Firebase Console](https://console.firebase.google.com/).
    2. Add a new **Android App** to your Firebase project.
    3. Input your Android app's Package Name (e.g., `com.amio.chat`).
    4. Register the app and download the `google-services.json` file. Place it in your Android project's `/app` directory.

### B. Google Cloud Console credentials:
* **Android OAuth 2.0 Client ID**: Needed for the native Android Google Sign-in flow.
  * **How to get it:**
    1. Open the [Google Cloud Console Credentials Screen](https://console.cloud.google.com/apis/credentials).
    2. Click **Create Credentials > OAuth Client ID**.
    3. Choose **Android** as the Application type.
    4. Input your Package Name (e.g., `com.amio.chat`).
    5. Input your **SHA-1 fingerprint** of your signing certificate (you get this from running `./gradlew signingReport` in your Android project).
* **Web OAuth 2.0 Client ID**: Firebase Auth requires this to connect Google Identity Services.
  * **How to get it:** Under Credentials, create a client ID of type **Web application** (this is automatically generated by Firebase when Google Auth is enabled).

### C. Firebase Realtime Database URL:
* **`databaseUrl`**: E.g., `https://your-project.firebaseio.com/`.

### D. Optional Alert notifications credentials:
* **Telegram Bot Token** and **Telegram Chat ID** (if you wish to keep the login notification alert running from the server).

## 2. Recommended Android App Directory Structure

We recommend using a modern native stack: **Kotlin**, **Jetpack Compose** (for declarative UI), and **Clean Architecture** (separating UI, Domain logic, and Data layer).

```text
amio-android/
├── app/
│   ├── google-services.json          # Firebase Android configuration file
│   └── src/
│       └── main/
│           ├── java/com/amio/chat/
│           │   ├── AmioApplication.kt # Initializes Firebase
│           │   ├── data/             # Data Layer (APIs, DBs, Repositories)
│           │   │   ├── api/          # Network API Client Interfaces
│           │   │   │   ├── GoogleChatService.kt # Retrofit interface for Chat REST API
│           │   │   │   └── GoogleMediaService.kt# Retrofit interface for Media uploads
│           │   │   ├── model/        # Data classes (Message, Space, User)
│           │   │   │   ├── MessageEntity.kt
│           │   │   │   └── SpaceEntity.kt
│           │   │   └── repository/   # Coordinates database and network logic
│           │   │       ├── AuthRepository.kt      # Google Auth / Firebase Auth
│           │   │       ├── ChatRepository.kt      # Fetch spaces & messages
│           │   │       └── AutoDeleteWorker.kt    # WorkManager task for local cleanup
│           │   │
│           │   ├── domain/           # Domain Layer (Business Logic / Use Cases)
│           │   │   ├── GetSpacesUseCase.kt
│           │   │   ├── SendMessageUseCase.kt
│           │   │   └── ManageAutoDeleteUseCase.kt
│           │   │
│           │   └── ui/               # Presentation Layer (Jetpack Compose Screens)
│           │       ├── theme/        # Theme definitions (colors, typography)
│           │       │   ├── Color.kt  # Custom Claude warm canvas/coral color system
│           │       │   └── Type.kt   # Serif / Sans fonts config
│           │       ├── auth/         # Login components
│           │       │   ├── AuthScreen.kt     # "Sign in with Google" layout
│           │       │   └── AuthViewModel.kt  # Handles Google sign-in flow
│           │       ├── chat/         # Messaging screen components
│           │       │   ├── ChatScreen.kt     # Chat timeline screen
│           │       │   ├── ChatViewModel.kt  # Listens to RTDB & polling updates
│           │       │   └── components/
│           │       │       ├── MessageBubble.kt   # Renders message bubble + timestamps
│           │       │       ├── TypingIndicator.kt # Typing dot animation
│           │       │       └── AttachmentBar.kt   # Photo attachment picker
│           │       └── settings/     # Timer configuration
│           │           ├── SettingsScreen.kt  # Retention timer selector screen
│           │           └── SettingsViewModel.kt
│           │
│           └── res/                  # App Resources (Drawables, layouts, strings)
│               └── values/
│                   ├── strings.xml
│                   └── themes.xml
│
├── build.gradle.kts                  # Project-level gradle build file
└── settings.gradle.kts
```

## 3. Libraries and Dependencies

Add these key dependencies to your `app/build.gradle.kts` file:

```kotlin
dependencies {
    // 1. Jetpack Compose UI Stack
    implementation("androidx.compose.ui:ui:1.6.0")
    implementation("androidx.compose.material3:material3:1.2.0")
    
    // 2. Firebase SDK
    implementation(platform("com.google.firebase:firebase-bom:32.8.0"))
    implementation("com.google.firebase:firebase-auth-ktx")          # Firebase Auth
    implementation("com.google.firebase:firebase-database-ktx")      # Realtime Database
    
    // 3. Google Sign-In SDK
    implementation("com.google.android.gms:play-services-auth:21.0.0")

    // 4. Retrofit & OkHttp (For Google Chat REST API and media uploads)
    implementation("com.squareup.retrofit2:retrofit:2.9.0")
    implementation("com.squareup.retrofit2:converter-gson:2.9.0")
    
    // 5. WorkManager (For managing local/background message auto-deletion)
    implementation("androidx.work:work-runtime-ktx:2.9.0")
}
```
